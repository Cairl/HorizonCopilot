### 添加

- **直接调整延迟**: 执行图光标停在可导航行上时，←→ 直接调整延迟（每次 ±10ms），Shift+←→ 以 1000ms 为单位调整，上下键划走即可离开，无需 Enter 进入编辑模式
- **分支紫色标签**: 执行图分支条件（有车状态 / 无车状态 / 抢车成功 / 抢车失败）改为紫色（MAUVE）显示
- **运行时界面统一**: 开始运行后保持 idle 三区域布局（开始按钮 + 执行图 + 特征库），通过开始按钮体现运行状态：绿色"开始运行"(未启动) / 黄色"等待运行"(游戏未聚焦) / 红色"停止运行"(运行中)，光标固定在按钮上按 Esc 随时取消，执行图锁定为只读并实时显示状态（✓ 已完成 / ▶ 当前 / ○ 等待 / · 未走分支置灰）
- **移除暂停界面**: 不再渲染单独的暂停/恢复界面和统计信息（尝试次数/发现次数），状态全部体现在开始按钮上
- **单实例锁**: 通过 PID 文件确保只允许一个实例运行，启动新实例时自动杀掉旧实例
- **执行图分支树**: 拍卖场抢车的执行图以树形结构展示完整分支（有车/无车、成功/失败），未走的分支整组置灰
- **动作类型标签**: 步骤类型用中文标签区分（点击/按下/抬起/等待/判断），为后续 press/release 动作预留
- **特征库命名同步**: 分支条件名（有车状态/无车状态/抢车成功/抢车失败）与特征库槽位标签一致

### 优化

- **节点标识符**: StepConfig/Branch 新增 `node_id` 字段用于运行时状态查找，替代旧的 dict 树 + key 映射方案
- **选中态视觉**: 执行图选中行改用背景高亮，开始运行按钮绿色加粗
- **横隔线留白**: 所有横隔线上下添加空行，视觉更清晰
- **延迟显示**: 点击步骤的延迟显示在按键左侧（`等待 50 ms  点击 ENTER`），表示点击前等待的时间

### 移除

- **旧运行渲染**: 移除 `render_running` / `_calc_running_width` / `_build_full_tree` / `_flatten_tree` 等基于 dict 树的运行时渲染代码，统一使用 `render_idle(running=True)`
- **判断标题行**: `match` 步骤不再渲染"判断"行，分支提升到 match 自身层级作为兄弟节点显示
- **行内编辑模式**: 移除 `_editing` / `_edit_step` / `_edit_ms` 状态、`_edit_step_delay()` / `_finish_inline_edit()` 方法，以及主循环中的编辑态分支，改为直接 ←→ 调整
