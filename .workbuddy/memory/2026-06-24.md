## 修复 Bug: SubaruTask 执行图运行时高亮缺失
- 根因：`_default_steps()` 用 `_KP`/`_CM` 辅助函数构造 65 个步骤，未设 `node_id`，导致 `_set(step.node_id, _ST_CUR)` 因 `node_id=None` 直接 return，运行时状态永远写不进 step，渲染时 `status == _ST_CUR` 始终 False，高亮失效。
- 修复：`_default_steps()` 末尾自动生成顺序 `node_id`（`subaru_s01`..`subaru_s65`）。同步验证 `_fill_node_ids` 可正常填补已保存 config 步骤的 node_id。
- 其他三个任务不受影响（默认步骤显式设了 node_id）。

## Shift+左右键「去零」逻辑修正
- 旧逻辑：`round(cur_ms / 1000) * 1000 + delta`，导致 1600 ms + Shift+Right → round(1.6)*1000 + 1000 = 2000 + 1000 = 3000
- 新逻辑：往移动方向截断到整千再步进。1600 + Shift+Right → floor(1600) + 1000 = 1000 + 1000 = 2000；1600 + Shift+Left → ceil(1600) - 1000 = 2000 - 1000 = 1000
- 改动位置：`core/task_base.py` `_adjust_step_delay` 第 786-789 行
