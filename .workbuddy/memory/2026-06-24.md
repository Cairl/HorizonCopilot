## 修复斯巴鲁任务鼠标点击不稳定

**问题**: `_win32_click_at` 中鼠标按下和抬起之间只有 10ms 间隔，游戏经常来不及注册点击，导致 click_match 步骤有时成功有时失败。

**修改** (`core/task_base.py`):

1. `_win32_click_at` 签名从 `(cx, cy, hover_delay)` 改为 `(cx, cy, button, click_delay)`:
   - `button`: 支持 left/right/middle，不再硬编码左键
   - `click_delay`: 鼠标按下到抬起的间隔，由 `step.delay` 传入（如 500ms），形成完整按下-保持-抬起周期
   - 悬停时间从 `hover_delay`(step.delay, 500ms) 缩短为固定 100ms（只需让游戏注册光标位置）

2. 三处调用点均更新为新签名，传入 `button=step.button` 和 `click_delay=step.delay`

## 循环次数公式与置信度步长

**循环次数公式显示与编辑** (`core/task_base.py` + 两任务子类):
- `BaseTask` 新增类属性 `loop_formula_template`（显示模板，如 `"{0}÷{1}"`）与 `loop_formula_default_terms`（默认各项值，`None`=无公式）；方法 `loop_formula_compute(terms)`（向下取整，子类覆写）
- **聚焦时**显示 `{N}={公式}`（如 `14=357÷30`、`32=(999−357)÷20`），高亮当前可编辑数字（下划线+黄）；**未聚焦只显示** `循环次数: N`（不显示公式）；运行态显示 `M/N`
- `Shift+←→` 切换当前可编辑数字（`_loop_active`：0=N, 1..n=terms；Shift+→ 下一个，Shift+← 上一个，循环）；普通 `←→` 调整当前高亮数字 ±1

- 各项下界 0；持久化 `loop_count` 与 `loop_formula_terms` 到 config
- 无公式任务（拍卖场/图外循环）Shift+←→ 无效，普通 ←→ 调 N
- 斯巴鲁公式 `{0}÷{1}` 默认 [357,30] → 11（357÷30=11.9 向下取整）
- 图内循环公式 `({0}−{1})÷{2}` 默认 [999,357,20] → 32（(999−357)÷20=32.1 向下取整）
- 公式项用 `string.Formatter().parse()` 拆分字面量与字段，渲染时高亮 active 对应项
- （2026-06-24 修正：原 Tab 切换改为 Shift+←→；原未聚焦显示公式改为只显示 N；原 Shift 步进10 改为切换数字）

**问题**: 从左侧栏进入右侧执行图，按 Up 聚焦循环次数行时，第一步骤行仍显示 `›`（双光标）。

## 循环次数运行时递减

**需求**：循环次数随每轮完成实质减少，减到 0 停止并归零（重置为无限）。

**实现** (`core/task_base.py` `_execute_tree` + `_build_loop_count_row`):
- 新增 `self._loop_infinite: bool` 实例属性，运行开始时 `= (self._loop_count == 0)`
- 循环条件改为 `while self._loop_infinite or self._loop_count > 0`（解决 0=无限 vs 0=完成的冲突）
- 每轮 `_walk` 完成（sig=="continue"）后：有限次数时 `self._loop_count -= 1` 并 `_save_loop_count` 持久化；减到 0 即 `return`（全部完成，`_loop_count` 已为 0=无限）
- 运行态显示：无限循环显示 `循环次数: 无限`；有限循环显示 `循环次数: {剩余}`（数字随每轮完成递减）
- 自然完成：`_loop_count` 已递减到 0，idle 显示「无限」（即「重置成无限」）
- 中途中断（Esc/失焦）：`_loop_count` 保留当前剩余值并已持久化，idle 显示剩余次数；下次运行从剩余值继续

## 用时格式 MM:SS + 实际/剩余重设计

**改动** (`core/task_base.py` `_steps_title` + `_execute_tree`):
- 新增 `_fmt_mmss(seconds)`：MM:SS 格式，MM 可超过 60（如 `61:01` 表示 61 分钟）
- `_steps_title` 从 HH:MM:SS 改为 MM:SS
- **实际用时**：不再显示当前循环进行中的时间，改为显示**上一次完整循环的耗时**（`_last_loop_time`），每轮 `_walk` 完成后记录 `time.monotonic() - _loop_start_time`；第一轮无上次数据时不显示实际行
- **预估剩余用时**：仅当有限循环（`_loop_infinite=False`）且剩余次数 > 0 且有实际数据时显示 `剩余: MM:SS`，算法 = `实际耗时 × 剩余次数`；无限循环不显示
- 运行态标题示例：`执行图 预计: 00:05 实际: 00:42 剩余: 05:53`
- `_last_loop_time` 在 `__init__`/`_init_nav` 中初始化为 `None`

**用时行移至 title 次行** (`core/task_base.py` `render_idle`):
- 用时信息由 `_steps_title`（title 字符串内）改为 `_build_timing_row`（独立行），`right_title` 简化为 `"执行图"`
- `render_idle` 中 `right_rows.insert(0, timing)` 将用时行插入到循环次数行之前
- `fixed_header` 从 1 行改为前 2 行（用时行 + 循环次数行），两者在视口滚动时均置顶
- `_build_steps_panel` 宽度采样追加 `"预计: 88:88 实际: 88:88 剩余: 88:88"` 确保用时行不溢出
- `_steps_title` 方法已移除
