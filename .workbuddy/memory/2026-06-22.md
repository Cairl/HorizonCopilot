## HorizonCopilot - 子界面两栏式大改动

### 改动范围
1. **子界面两栏式（书页布局）** — `core/task_base.py` `render_idle` 重写为两栏：左栏菜单（开始运行 / 设置分节标题 / 执行图 / 特征库），右栏常开，默认展开「执行图」。`Tab` 切换左右栏焦点。新建 `left_nav`/`right_nav`/`col_focus`/`right_view` 导航模型取代旧的单一扁平 `self.nav`。
2. **执行图按 Enter = 从该行开始运行** — 新增 `_start_node`，`_handle_right_key` 在步骤行按 Enter 时设置 `_start_node=step.node_id` 并启动。通用执行器 `_execute_tree(start_node_id)` 首轮跳过前缀节点（含进入包含 start 的分支）。
3. **特征缺失不阻碍运行** — `_can_start` 只要求 store 存在（不再要求所有模板就绪）。`_match_step` 执行前检查 `feature_type` 模板是否就绪，缺失则设 `_missing_msg` 并返回 "stop" 停止运行；消息在 idle 首帧显示，按键后清除。
4. **拍卖场抢车固定第一序列** — `tasks/__init__.py` `discover_tasks` 把 `tag=="auction"` 钉在首位，其余按目录名排序（新任务默认出现在下方）。

### 关键架构：通用树执行器 `_execute_tree`
- 取代两个任务里手写的 `_run_core_loop` 主循环（sniper/racer 现在只剩预处理钩子 + 调 `_execute_tree`）。
- 处理 `keypress`/`hold`/`wait`/`match`（分支型：截图后取匹配分支，无匹配走 `fallback_key`；无分支型：循环截图直到识别到）。
- `branch.loop=="结束"` → END（`stats["found"]++`）；其他 → CONTINUE。
- 通用 dimming：`_dim_other_branches`/`_dim_all_branches` 通过 `_subtree_node_ids` 计算，等价于旧的手写 `_DIM_*` 集合。
- `StepConfig` 新增 `fallback_key` 字段（sniper 的 match_car/match_result 设 `"esc"`），序列化往返已验证。

## 交互调整: 下划线 + ←→切栏 + Enter编辑

### 改动
- **打开标记**: 执行图/特征库标签用 UNDERLINE 表示当前打开（不再 TEAL）。UdlrTui colors.py 新增 `C.UNDERLINE`。
- **列切换**: Tab 移除。→ 在左栏 = 进入右栏；← 在右栏(非编辑态) = 返回左栏。
- **编辑模式** (`right_editing`): 右栏按 Enter 进入编辑态。
  - 步骤: ←→ 调延迟(±10ms, Shift ±1000ms), Up/Down 移动(保持编辑), Enter = 从该行运行, Esc = 退出编辑。
  - 特征: ←→ 切截取/删除, Enter = 执行, Esc = 退出。
  - 非编辑态: 步骤行无括号、延迟无下划线；特征行无括号。
  - 编辑态: 步骤延迟 BOLD+UNDERLINE+YELLOW；特征括号 `[截取]`/`[删除]` 出现。
- 0ms 暂停模式直接进入编辑态(`right_editing=True`)，←→ 直接调延迟。
- Esc 分层: 右栏编辑态 → 退出编辑(留右栏); 右栏非编辑 → 返回左栏; 左栏 → 退出任务。

## 图外循环赛事 - 检测赛事准备增加「降低难度」分支

### 需求
赛事准备界面偶发弹出「降低难度」选项，需要自动处理：检测到时按 ↓ + Enter 关闭，然后回到赛事准备继续检测。

### 改动
1. **`core/task_base.py`** — 扩展 match 步骤支持循环分支判断：
   - `StepConfig` 新增 `loop_until_match: bool = False` 字段（序列化往返已验证）。
   - `_match_step` 拆分为三种派发：无分支（循环检测）、`_oneshot_branched_match`（原单次分支逻辑，拍卖场用）、`_looping_branched_match`（新循环分支）。
   - `_looping_branched_match`：每次倒计时后依次检测各分支特征，取首个匹配者执行；无匹配时重新倒计时。分支 `loop=="回到本步骤"` 回到本 match 继续；`"结束"` 结束运行；其他值继续后续步骤。
   - `Branch.loop` 文档补 `"回到本步骤"` 说明。

2. **`tasks/图外循环蓝图赛事/racer.py`** — 检测赛事准备改为循环分支：
   - `FeatureType` 新增 `RACE_LOWER_DIFFICULTY = "race_lower_difficulty"`，`SLOT_LABELS` 新增 `"降低难度"`。
   - `_default_steps()` / `_DEFAULT_STEPS` 的 `match_prep` 步骤改为 `loop_until_match=True` + 两分支：`降低难度`（↓+Enter, loop="回到本步骤"）/ `赛事准备`（空, loop=None 继续）。
   - 顺手修复 `_default_steps()` 中重复的 `enter6` bug（原有两个连续 node_id="enter6"，删掉一个，与 config 和 docstring 对齐）。
   - `feature_type` 保持 `"race_prep"`（不含 `race_lower_difficulty`），降低难度为可选特征——未截取时分支永不匹配，任务按原流程运行。

3. **`tasks/图外循环蓝图赛事/data/config.json`** — slots 新增 `race_lower_difficulty: null`；match_prep 步骤改为带 `loop_until_match: true` + branches 结构，保留用户自定义 `delay: 1.0`。

4. **`AGENTS.md`** — 执行图模型从两种结构改为三种（新增循环分支判断）；match 步骤类型说明补三模式；特征库章节赛事任务槽位数从 1 改为 3。

### 验证
- JSON 格式有效；两个 py 文件 py_compile 通过。
- config 加载 → `_step_from_dict` → `match_prep.loop_until_match=True`、两分支结构正确。
- `_step_to_dict` 往返序列化保留 `loop_until_match` 和 branches。
- `_fill_node_ids` 对齐正常（loaded 与 default 结构一致，16 步）。
- `_build_node_map` 产出 20 节点（16 顶层 + 2 分支 + 2 子步骤）。
- `_iter_nav_steps` 产出 18 可导航节点（分支条件行不可导航，子步骤可调延迟）。
- 拍卖场任务不受影响：`match_car.loop_until_match=False`（one-shot 保留），`fallback_key="esc"` 保留。

### 设计要点
- `feature_type` 不含 `race_lower_difficulty`，所以缺失特征检查只要求 `race_prep`。降低难度槽位未截取时 `match_slot` 返回 `(None, 0.0)`，分支自然不匹配，任务降级为原流程。
- `_looping_branched_match` 每轮开头调 `_reset_runtime_status([step])` 重置子树状态，让用户看到 match 步骤重新激活进入倒计时。
- 循环分支模式不使用 `fallback_key`（无兜底按键，无匹配时直接重试）。
