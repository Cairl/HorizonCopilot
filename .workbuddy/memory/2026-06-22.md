## HorizonCopilot - 子界面两栏式大改动

### 改动范围
1. **子界面两栏式（书页布局）** — `core/task_base.py` `render_idle` 重写为两栏：左栏菜单（开始运行 / 设置分节标题 / 执行图 / 特征库），右栏常开，默认展开「执行图」。`Tab` 切换左右栏焦点。新建 `left_nav`/`right_nav`/`col_focus`/`right_view` 导航模型取代旧的单一扁平 `self.nav`。
2. **执行图按 Enter = 从该行开始运行** — 新增 `_start_node`，`_handle_right_key` 在步骤行按 Enter 时设置 `_start_node=step.node_id` 并启动。通用执行器 `_execute_tree(start_node_id)` 首轮跳过前缀节点（含进入包含 start 的分支）。
3. **特征缺失不阻碍运行** — `_can_start` 只要求 store 存在（不再要求所有模板就绪）。`_match_step` 执行前检查 `feature_type` 模板是否就绪，缺失则设 `_missing_msg` 并返回 "stop" 停止运行；消息在 idle 首帧显示，按键后清除。
4. **拍卖场抢车固定第一序列** — `tasks/__init__.py` `discover_tasks` 把 `tag=="auction"` 钉在首位，其余按目录名排序（新任务默认出现在下方）。

### 关键架构：通用树执行器 `_execute_tree`
- 取代两个任务里手写的 `_run_core_loop` 主循环（sniper/racer 现在只剩预处理钩子 + 调 `_execute_tree`）。
- 处理 `keypress`/`hold`/`wait`/`match`（分支型：截图后取匹配分支，无匹配走 `fallback_key`；无分支型：循环截图直到识别到）。
- `branch.loop=="结束"` → END（`stats["found"]++`）；其他 → CONTINUE。
- 通用 dimming：`_dim_other_branches`/`_dim_all_branches` 通过 `_subtree_node_ids` 计算，等价于旧的手写 `_DIM_*` 集合。
- `StepConfig` 新增 `fallback_key` 字段（sniper 的 match_car/match_result 设 `"esc"`），序列化往返已验证。

## 交互调整: 下划线 + ←→切栏 + Enter编辑

### 改动
- **打开标记**: 执行图/特征库标签用 UNDERLINE 表示当前打开（不再 TEAL）。UdlrTui colors.py 新增 `C.UNDERLINE`。
- **列切换**: Tab 移除。→ 在左栏 = 进入右栏；← 在右栏(非编辑态) = 返回左栏。
- **编辑模式** (`right_editing`): 右栏按 Enter 进入编辑态。
  - 步骤: ←→ 调延迟(±10ms, Shift ±1000ms), Up/Down 移动(保持编辑), Enter = 从该行运行, Esc = 退出编辑。
  - 特征: ←→ 切截取/删除, Enter = 执行, Esc = 退出。
  - 非编辑态: 步骤行无括号、延迟无下划线；特征行无括号。
  - 编辑态: 步骤延迟 BOLD+UNDERLINE+YELLOW；特征括号 `[截取]`/`[删除]` 出现。
- 0ms 暂停模式直接进入编辑态(`right_editing=True`)，←→ 直接调延迟。
- Esc 分层: 右栏编辑态 → 退出编辑(留右栏); 右栏非编辑 → 返回左栏; 左栏 → 退出任务。
