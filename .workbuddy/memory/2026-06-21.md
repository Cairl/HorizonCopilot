# 2026-06-21

## HorizonCopilot - 拍卖场抢车任务运行状态修复

### 问题
用户报告：点击"开始运行"后游戏窗口正确切换，但 UI 仍显示 idle 状态（绿色"开始运行"），且没有任何后续操作执行。

### 根因
两个相关问题：

1. **`handle_idle_key` 未检查 `activate_game_window()` 返回值** — 即使未找到游戏窗口也会设置 `self.state = "running"`，进入假的运行态。

2. **`_run_core_loop` while 循环首个 `guard.check()` 时序问题** — `activate_game_window()` 调用 `SetForegroundWindow` 将游戏切换到前台，但 Windows 的窗口切换是异步的。`_run_core_loop` 中的 setup 代码（drain_keyboard、build_node_map、calc_width）全部是进程内操作，微秒级完成。当 while 循环的 `guard.check()` 被调用时，游戏可能尚未完全成为前台窗口。`is_game_focused()` 返回 False，`_run_core_loop` 立即 return。`run()` 方法重置状态为 idle，重新渲染绿色"开始运行"。

### 修复
- `core/task_base.py` L403-408: `handle_idle_key` 中检查 `activate_game_window()` 返回值，失败时拒绝进入 running 状态。
- `tasks/auction/sniper.py` L383-392: `_run_core_loop` 在进入 while 循环前添加焦点重试机制（最多 10 次，间隔 50ms，总计 500ms），确保游戏窗口完全切换到前台后再开始执行。
